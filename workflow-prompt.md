## 任务概述

## 工作流步骤（按顺序执行）

### 第一步：扫描各平台最新发布
依次访问上述每个URL，提取最近发布的研究内容：
- 论文标题
- 发布日期
- 简要描述/摘要
- 相关链接（arXiv链接、GitHub链接等）

**判断标准**：只关注最近7天内的新发布内容。如果没有新内容，记录"无更新"并继续下一个平台。

### 第二步：获取论文详情
对于每篇识别到的新论文/技术报告：
1. 如果有arXiv链接 → 直接访问获取完整信息
2. 如果是GitHub技术报告 → 检查README中的论文链接
3. 如果是博客文章 → 检查文中是否引用了arXiv论文
4. 在arXiv上搜索论文标题确认是否有预印本

需要提取的信息：
- 论文完整标题
- 所有作者姓名及其单位（affiliation）
- arXiv编号（如有）
- 摘要
- 关键贡献点

### 第三步：识别高校合作
检查每篇论文的作者列表和单位信息：
- 是否有高校（大学/研究院）参与？
- 如果**没有高校参与**：
  - 记录论文核心内容和关注点
  - 该论文分析到此结束
- 如果**有高校参与**：
  - 记录高校名称
  - 记录来自高校的作者姓名
  - 继续第四步

### 第四步：高校人员信息调查
对于每位来自高校的作者：
1. 搜索 "[作者姓名] [高校名称]" 了解其身份
2. 判断是否为在读学生（博士生/硕士生）
3. 如果**不是在读学生**（教授/研究员等）：
   - 记录其职位和研究方向
   - 到此结束
4. 如果**是在读学生**：
   - 继续第五步

### 第五步：学生详细信息追踪
对于识别为在读学生的作者：
1. 搜索其个人主页（Google Scholar / 个人网站 / LinkedIn）
2. 确认其：
   - 所在院系/实验室
   - 导师姓名
   - 研究方向
   - 已发表论文列表
   - 预计毕业时间（如有）
3. 搜索其导师信息：
   - 导师个人主页
   - 实验室网站
   - 实验室研究方向
   - 实验室其他成员

```markdown
|------|------|-----------|
| ... | 有更新/无更新 | N |

```

## 注意事项
- 如果某个网站无法访问，记录错误并继续其他网站
- 优先处理最新发布的内容
- 对于中文论文/博客，保持中文输出
- 高校判断：包括但不限于 清华、北大、中科院、浙大、上交、MIT、Stanford、CMU、Berkeley等
- 学生判断线索：个人主页标注"PhD student"/"博士生"、Google Scholar显示在读等
- **⚠️ 姓名规范（严格遵守）**：当无法通过可靠来源（个人主页、LinkedIn、Google Scholar 等）确认作者的中文名时，**必须直接使用拼音或英文名，严禁编造中文名**。论文署名通常是拼音（如 "Yiming Zhang"），不得擅自猜测对应中文字。只有在可验证来源明确写了中文名时才使用中文名。
